Recipe Manager

Application Requirements

Apache Web Server Requirements
1)	Version 2.2 or better
2) 	mod_rewrite
3) 	.htaccess
4) 	vhost


PHP Requirements
1) 	Version 5.2 or better
2) 	cURL module



(See Apache installation notes below)



CakePHP Installation
CakePHP is an MVC framework for PHP. (http://cakephp.org/)

1) 	Get the latest version of the CakePHP libraries. You can get them from our repository with
	svn export svn://10.1.2.50/eti_web_projects/cake_1.2.5 _libraries
	Or download them from the website. Delete the app directory if its present in the download.
2) 	Place them in a folder called _libraries next to the solotech install directory. 
	Use the same naming conventions as shown.

	solotech/
	_libaries/
		cakePHP/
			cake/
			vendors/
			.htaccess
			index.php
			README
		


Application Installation

1) 	Get the latest version of the software
2) 	Place into a folder so that structure is like the following:
	
	recipe_manager/
		app/
		.htaccess
		index.php
		INSTALL.txt (this file)
		README
		
	WARNING: DO NOT MODIFY ANY APPLICATION FILES EXCEPT WHERE NOTED IN THE DOCUMENT BELOW
		
3) 	Give Apache read and write permissions to: solotech/app/tmp and all its sub directories
4) 	Setup an Apache virtual host that points to: solotech/app/webroot
5) 	Setup a DNS entry to point to the application




Apache Web Server and PHP Setup

This only documents what is required for SoloTech to run and is not an Apache tutorial. 
Please see http://httpd.apache.org/docs/2.2/ for Apache documentation.
Please see http://www.php.net/manual/en/ for PHP documenation

http://httpd.apache.org/docs/2.2/install.html
Apache 2.x with SSL, PHP5, ReWrite
	./configure --enable-ssl=shared --with-ssl=/usr/local/ssl/ --enable-rewrite=shared --enable-setenvif --enable-so

http://www.php.net/manual/en/install.unix.apache2.php
PHP 5.2.x with SOAP, MySQL, SSL, cURL and Apache
	./configure --with-apxs2=/usr/local/apache2/bin/apxs --with-mysql --with-curl --enable-soap --with-openssl=/usr/local/ssl
